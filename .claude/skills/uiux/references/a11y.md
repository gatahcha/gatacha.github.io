# Accessibility checklist

Run through this for every component. A11y is not optional polish — it's part of
"pro". Ship none of the items below broken.

## Semantics

- Headings are real `<h1>`…`<h6>`, in order, one `<h1>` per route. Don't fake
  titles with `<div className="text-4xl font-bold">`.
- Buttons do actions, links navigate. A `<div onClick>` is always wrong — use
  `<button>` (gets keyboard + Enter/Space for free).
- Lists = `<ul>`/`<li>`. Tables only for tabular data.
- Decorative icons: `aria-hidden`. Meaningful icons need a text label or
  `aria-label`.

## Focus

- Every interactive element is keyboard-reachable and has a visible focus ring.
  Don't remove `outline` without replacing it: `focus-visible:ring-2
  focus-visible:ring-accent focus-visible:ring-offset-2
  focus-visible:ring-offset-background outline-none`.
- Use `focus-visible` (not `focus`) so the ring shows for keyboard users but not
  on mouse click.
- Tab order matches visual order. Don't set positive `tabindex` — let DOM order
  do it.

## Color & contrast

- Body text on dark `#0a0a0a`: use `text-white/70` minimum (≈ AA at 16px).
  Tertiary `text-white/50` only for large or non-essential text.
- Interactive states must not rely on color alone — add underline, icon, or
  weight change so colorblind users see the state.
- The mint accent `#10E8CC` on dark passes for large text / UI; don't use it
  for body copy (it's a light color, hard to read in long runs).

## Motion

- `MotionConfig reducedMotion="user"` at the app root (see `motion.md`).
- `motion-safe:`/`motion-reduce:` on CSS transitions for the things that
  matter.
- No infinite autoplay loops beside readable content.

## Targets

- Tap targets ≥ 44×44px (`min-h-11 min-w-11` on mobile). Pills in `skillcard`
  are fine for chips but nav links and buttons need real targets.

## Images

- `next/image` with `unoptimized` is fine (static export). Always provide
  meaningful `alt`; `alt=""` only when the image is decorative and adjacent text
  already describes it.
- Avoid text-in-image; if text matters, render it as text.

## Forms (when added)

- Every input has a `<label>` (real `<label htmlFor>`, not a placeholder
  masquerading as one).
- Error messages are `aria-describedby` and announced (`role="alert"` or
  `aria-live`).
- Required fields marked with both `*` and `aria-required`.

## Reduced data & perf

- No autoplaying video/audio. No flashing > 3Hz (photosensitivity).
- Images sized to display dimensions to avoid layout shift (set `width`/
  `height` or use `aspect-*`).
